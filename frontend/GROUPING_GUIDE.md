# GraphInsight 节点分组功能使用指南

## 功能概述

GraphInsight 的节点分组功能让您可以将图谱中的节点按照类型或自定义规则进行分组，提供更清晰的数据组织和可视化体验。

## 快速开始

### 1. 访问分组面板
1. 打开 GraphInsight 应用 (http://localhost:5173)
2. 在左侧控制面板中点击"分组"标签页
3. 您将看到分组管理界面

### 2. 自动按类型分组
1. 首先执行一个查询获取图谱数据：
   ```cypher
   MATCH p=()-[]->() RETURN p LIMIT 25
   ```
2. 在分组面板中开启"按类型自动分组"开关
3. 系统会自动为每种节点类型创建一个分组
4. 每个分组使用不同的颜色进行区分

### 3. 手动创建分组
1. 点击"创建分组"按钮
2. 输入分组名称（如"重要节点"）
3. 选择要包含的节点类型（可多选）
4. 选择分组颜色
5. 点击"创建"完成

## 分组交互

### 折叠和展开分组
- **双击分组节点**: 在图谱中双击分组可切换折叠/展开状态
- **折叠状态**: 分组显示为紧凑的圆角矩形，子节点被隐藏
- **展开状态**: 分组显示为虚线边框，包围所有子节点

### 分组管理
- **编辑分组**: 点击分组列表中的菜单按钮 → 编辑分组
- **删除分组**: 点击分组列表中的菜单按钮 → 删除分组
- **清除所有**: 点击"清除所有分组"按钮

### 视觉提示
- **鼠标悬停**: 悬停在分组节点上显示详细信息
- **颜色区分**: 每个分组使用独特的颜色主题
- **状态指示**: 分组列表显示节点数量和折叠状态

## 使用技巧

### 1. 数据探索
- 使用自动分组快速了解图谱的节点类型分布
- 折叠不关心的分组，专注于重要数据
- 通过颜色快速识别不同类型的节点

### 2. 复杂图谱管理
- 对于大型图谱，先按类型分组再进行分析
- 使用手动分组将相关节点组织在一起
- 结合过滤功能隐藏不需要的分组

### 3. 演示和报告
- 折叠次要分组，突出重点内容
- 使用有意义的分组名称提高可读性
- 选择合适的颜色搭配增强视觉效果

## 高级功能

### 分组样式定制
- 每个分组可以设置独特的颜色
- 分组标签可以通过"显示分组标签"开关控制
- 分组样式遵循 Neo4j Browser 的专业设计标准

### 状态持久化
- 所有分组配置自动保存到浏览器本地存储
- 页面刷新后自动恢复分组状态
- 分组的折叠状态也会被记住

### 布局集成
- 分组状态变化时自动触发布局重排
- 支持所有现有的布局算法
- 分组节点参与布局计算，确保美观的排列

## 实际应用场景

### 1. 知识图谱分析
```cypher
MATCH (n) RETURN n LIMIT 50
```
- 按实体类型分组（人物、地点、事件等）
- 折叠不相关的类型，专注于特定领域

### 2. 社交网络分析
```cypher
MATCH (p:Person)-[r:KNOWS]->(f:Person) RETURN p, r, f LIMIT 30
```
- 按人物属性分组（部门、地区、角色等）
- 观察不同群体之间的连接模式

### 3. 业务流程图
```cypher
MATCH (s:Step)-[r:NEXT]->(n:Step) RETURN s, r, n
```
- 按流程阶段分组
- 折叠已完成的阶段，关注当前进度

## 🚨 注意事项

### 性能考虑
- 大型图谱（>500节点）建议先使用过滤功能减少节点数量
- 复杂分组可能影响布局计算时间
- 建议合理使用折叠功能减少渲染负担

### 最佳实践
- 分组名称应该简洁明了
- 避免创建过多的小分组
- 定期清理不需要的分组
- 结合其他功能（过滤、样式）获得最佳体验

## 🔗 相关功能

- **节点样式配置**: 在详情面板中配置节点外观
- **布局算法**: 在布局面板中选择合适的排列方式
- **过滤功能**: 在过滤面板中隐藏不需要的节点
- **导出功能**: 导出包含分组的图谱图片
