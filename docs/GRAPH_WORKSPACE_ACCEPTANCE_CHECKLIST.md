# Graph Workspace Acceptance Checklist

更新日期：2026-06-28

## 自动回归

提交前至少执行：

```bash
cd frontend
npm run build
npm run e2e:workspace
```

若没有 `ADMIN_TOKEN` / `E2E_ADMIN_TOKEN`，`e2e:workspace` 会按设计跳过登录态断言；发布验收应通过 `frontend/tests/run_admin_e2e.sh` 注入真实管理员 token 后执行。

## 工作区布局

- `/workspace?graph_demo=1` 能进入图谱工作区并显示演示图谱数据。
- 宽度 1280px 下右侧面板必须可见，不能被响应式规则隐藏。
- 右侧面板至少展示 `详情 / 统计 / 分析 / 控制` 四个入口。
- 左侧问答面板、中间画布、右侧详情面板之间不能互相覆盖。

## 2D 图谱

- 默认 2D 画布可见，能显示节点和关系。
- 点击节点后，相关节点和关系应明显加深，非相关元素保持弱化。
- 节点不应出现黑色硬轮廓；节点视觉应保持柔和。
- 查询结果加载后不应自动把所有节点当搜索命中高亮。

## 3D 图谱

- 点击 `切换到 3D` 后，3D WebGL 容器可见，右侧面板仍保持可见。
- 大规模图谱不应压成单一平面，应能看到 z 轴层次和不同类型节点的空间分布。
- 默认只显示关键节点标签，避免满屏文字遮挡。
- hover 节点时，应显示当前节点和少量重要邻居标签，并加深相关关系线。
- click 节点时，应显示当前节点和一圈重要邻居标签；完整属性通过 hover tooltip 和右侧 `节点详情` 查看。
- 切回 2D 后，2D 画布和右侧面板仍正常。

## 已知边界

- 3D 不支持 SVG 导出；需要 SVG 时切回 2D。
- 3D 标签不是全量常驻能力，默认铺满所有节点名称会降低大图可读性。
- 工作区 E2E 只做结构与模式切换 smoke；截图级视觉验收仍需要人工确认。
